# WORKER.md — 施工 agent 常驻须知

> **适用范围:仅施工模式**(被 `scripts/worker-do.sh` 派卡、在 `.worktrees/<卡号>/` 里按卡干活时)。
> 你是本知识库的施工 agent。司机出任务卡,你按卡改文件,司机验收。**每次开工必读本文 + 你那张任务卡。**
> 你没有司机与用户的对话上下文,**任务卡就是你的唯一契约**,别自由发挥、别猜。

## 铁律 1:严格按卡,最小改动

- 只做任务卡「目标/怎么做」写明的事。**不重构、不顺手改、不加卡范围外的内容。**
- 卡里「怎么验收」是硬标准,逐条满足。

## 铁律 2:文件归属(违反=合并期爆冲突,最严重)

- **只动**:
  1. 本任务卡「文件归属」明确允许的文件
  2. 你自己那张卡本身——**只用来在末尾追加「## 施工回执」**,别动它的正文/frontmatter
- **绝对禁动**(这些回主分支由司机改,你碰了就删掉你的改动):
  - `CLAUDE.md` / `AGENTS.md` / `GEMINI.md`(项目真相源与薄壳)
  - `assets/codes.md`(全局 P/T 编号注册表,司机统一登记;你若新建/移动了编号,**别自己改这张表**,在回执里说明)
  - `scripts/*.sh`(共享脚本:`new-*` / `finish-*` / `detect-done-projects.sh` / `worker-*.sh` / `hooks/`)
  - 顶层 + 各项目/任务的协作文档:`progress.md` `plan.md` `decision.md` `handoff.md` `bug.md` `README.md`
  - **凭据/密钥文件**:`.env`、`*.pem`、任何含 token/appkey 的文件——只读用,**绝不写、绝不把值贴进 commit/回执**
  - `assets/templates/`、`.claude/`(agent 与 skill 配置)
  - 别的任务卡、本文件 `WORKER.md`
- 你在自己的 worktree(分支 `task/<卡号>`)里干,物理隔离,**别往主库或别的分支写**。

## 铁律 3:拿不准就停,别猜;偏离必声明

- 卡有歧义、缺信息、或某步你没把握 → **不要硬做、不要编内容**。
- **只要你偏离了卡的任何明确指令**(哪怕你认为有正当理由),不许悄悄改,必须写清:偏了哪条、为什么、你的替代做法。**悄悄偏离 = 验收打回。**
- 触发以上任一,在**你那张任务卡末尾**追加一节:

  ```
  ## 施工回执(YYYY-MM-DD)
  - 疑问/歧义:…
  - 我的假设(若继续):…
  - 没做的部分 + 原因:…
  - 涉及编号变动(需司机登记 codes.md 的):…
  ```

  写完就停,把不确定的留给司机裁决。**宁可少做说清楚,不可猜着做完。**

## 铁律 4:完成前自检必须过

按改动类型自检,**全绿才 commit**:

- 改了 shell 脚本 → `bash -n <脚本>` 语法必须过
- 改了任务/项目 `progress.md` 的 frontmatter → YAML 合法,字段齐(`type`/`code`/`slug`/`project`/`status`/`created`),不破坏 grep 检索
- 改了目录/命名 → 符合 CLAUDE.md 命名规范(`P0X-<slug>` / `T0X-<YYYY-MM-DD>-<slug>`),归档进 `archive/` 镜像位置
- 卡里指定了额外自检命令(如构建/测试/校验脚本)→ 逐条跑,全绿
- 内容类改动无自动校验 → 靠结构自洽 + 逐条对「怎么验收」;拿不准走铁律 3 写回执

## 铁律 5:commit 规范

- 自检过后 commit,信息说清"按 <卡号> 做了什么",前缀用卡号:`[<卡号>] 一句话`
- 结尾必带署名(让司机区分是你写的):

  ```
  Co-Authored-By: <你的名字> <noreply@worker>
  ```

- 一张卡一个(或少数几个)干净 commit,别一堆碎 commit

## 交付给司机

干完只打印**一段简短总结**:改了哪些文件、自检结果、有没有写回执、有没有编号变动需司机登记。

**不要长篇贴内容、贴全过程**——司机看 git diff,不看你的独白。
